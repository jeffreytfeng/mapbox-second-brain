# My Voice Profile Builder

## Goal

Understand and replicate the user's preferred communication style across the channels they use — long-form strategy docs, stakeholder update emails, and Slack — by analyzing real past examples. The resulting profile at `Knowledge/Context/my-voice.md` should be specific enough that any future draft (PRFAQ, weekly update, Slack message) sounds like the user wrote it.

## Instructions

### Step 1: Check for an existing profile

- Read `Knowledge/Context/my-voice.md` if it exists.
- If it exists: ask the user whether to **update** (full re-analysis, overwrite) or **append** (add new observations without erasing existing ones). Default to **update** if they say "analyze my writing" or "learn my voice."

### Step 2: Auto-gather writing samples

Do NOT ask the user to paste samples manually. Pull them automatically:

**Google Drive MCP** — search for docs the user authored:
- `mcp__claude_ai_Google_Drive__search_files` with queries like `"PRFAQ"`, `"strategy"`, `"1-pager"`, `"roadmap"`, `"weekly update"` filtered to the user's email
- Also search for titles that look like authored documents (not meeting notes auto-generated by AI assistants)
- Read up to 3 of the most substantive results via `read_file_content`

**Gmail MCP** — find emails the user wrote (not received):
- Search `from:<user-email> newer_than:60d` — focus on threads where the user authored the opening message
- Prefer: weekly updates, stakeholder summaries, async decision posts, OKR updates
- Pull 3–5 representative emails

**Slack MCP** — find messages the user sent:
- Search `from:me newer_than:30d` in key channels the user belongs to (read from `Knowledge/Context/team-context.md` or ask if unclear)
- Also search `from:me` for longer strategic messages
- Pull 5–10 messages that are more than 2 sentences (skip one-liners and emoji reactions)

**Raw/ fallback** — if MCP sources return nothing or are unavailable:
- Scan `Raw/` for any files containing the user's writing (weekly update drafts, meeting notes they authored, `Drafts/`)
- Read `Drafts/` for past weekly updates

Aim for at least **8 samples total** across at least **2 channels**. If you can't reach 8, proceed with what you have and flag the gap.

### Step 3: Analyze style across channels

For each channel (long-form docs, email, Slack), extract:

**Tone**
- Formal vs. conversational register
- Data-driven vs. narrative (does the user lead with numbers or context?)
- Direct (bottom-line-up-front) vs. build-up (context-first)
- How confident/opinionated vs. hedge-y

**Structure**
- Prose vs. bullets — what ratio?
- Does the user use TL;DRs or exec summaries?
- Numbering style (1. vs. - vs. bold headers)
- How long are paragraphs? Are they punchy or discursive?
- Does the user use tables?

**Formatting**
- Header usage (##, bold, or none?)
- Capitalization style (title case headers vs. sentence case?)
- Emoji usage — if so, where and how much
- Bold for emphasis — how frequently and for what?
- Em dashes, parentheticals, ellipses

**Vocabulary & Phrases**
- Recurring power phrases or framing devices ("the question is...", "this matters because...", "here's what I'd watch:")
- Jargon tolerance — does the user define terms or assume shared context?
- Favorite qualifiers ("roughly", "likely", "at risk")
- Sign-offs and salutations in email

**Channel differences**
- How does tone shift between a PRFAQ and a Slack message?
- How formal are emails vs. Docs?
- Does the user write differently to different audiences?

### Step 4: Build the profile

Create or overwrite `Knowledge/Context/my-voice.md` with this structure. Fill each section based on **observed patterns from the samples** — do not use defaults if you have real data. Flag any section where you had insufficient samples.

```markdown
# My Voice Profile
*Last updated: [date] | Sources: [N Drive docs, N emails, N Slack messages]*

## Global Tone

[5 bullets max — the cross-channel rules that always hold]

## Long-Form Docs (PRFAQs, Strategy Memos, 1-Pagers)

- **Structure:** [observed pattern]
- **Opening:** [how the user starts — TL;DR, context, question?]
- **Data usage:** [leads with numbers or narrative?]
- **Headers:** [style observed]
- **Length norms:** [concise vs. thorough?]

## Stakeholder Updates & Email

- **Format:** [bullets or prose? structure observed]
- **Lead:** [what comes first — status, ask, or context?]
- **Salutation:** [observed openers]
- **Sign-off:** [observed closers]
- **Tone:** [register shift vs. Slack/docs]

## Slack

- **Length:** [typical message length]
- **Tone:** [casual/emoji/direct/etc.]
- **Formatting:** [bullets? inline bold? plain text?]
- **How they ask for things:** [observed patterns for cross-functional asks]
- **Escalation style:** [how they flag urgency]

## Phrases to Use

[Verbatim recurring phrases or framing devices observed — up to 10]

## Things to Avoid

[Anti-patterns not in their writing that AI tends to introduce — up to 5]
```

### Step 5: Confirm and save

- Show the user a **summary** of the key findings (3–5 bullets) before saving — not the full profile.
- Ask: "Does this feel accurate? Anything to adjust before I save?"
- On confirmation, write to `Knowledge/Context/my-voice.md`.
- Update the `*Last updated:*` line to the current datetime (format: `HH:MM PDT, Weekday DD Month YYYY`).

## Constraints

- Never fabricate style observations — if evidence is thin, say "insufficient samples; using placeholder."
- Cap each section at 5 bullets for maintainability.
- If Drive/Gmail/Slack return nothing, ask the user to paste 2–3 samples directly rather than proceeding blind.
- Do not include raw quotes from the user's writing in the profile unless they illustrate a specific, non-obvious pattern.
